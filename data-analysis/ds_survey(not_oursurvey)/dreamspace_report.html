🔍 Analytical deep-dives
A. Weekly response trend
How this was calculated: Grouped by ISO week number derived from start timestamp. Each bar = total submissions in that calendar week.
B. Completion time distribution
How this was calculated: Duration = completion − start time. Responses > 10 min excluded (n=0).
C. Gender × coding intent (PO students)
How this was calculated: Cross-tab of PO gender vs 'want to code more'. % computed within each gender group.
D. Enjoyment vs knowledge gap by workshop
How this was calculated: MEAN(enjoyment) and MEAN(knowledge gain) per workshop type. Gap = enjoyment − knowledge. Only workshops with n≥5 shown.
E. Location effect — multi-metric comparison
How this was calculated: MEAN of enjoyment, instructor rating, and self-pride per location. Only rows with all three non-null included.
F. Prior coding experience → post-workshop intent
How this was calculated: Cross-tab of prior coding experience vs want to code more. % computed within each prior-experience group.
G. Top enjoyment drivers — weighted frequency
How this was calculated: Multi-select field split by ';'. Each tag counted independently. % = tag count / total respondents × 100.
H. OBA/TUMO conversion funnel
How this was calculated: Sequential count of qualifying responses within OBA Next location subset. Each stage = affirmative responses.
I. Satisfaction radar scorecard
How this was calculated: Each axis = MEAN of corresponding Likert column. Scale 0–5.
J. Age distribution across education levels
How this was calculated: PO and VO age columns combined into one distribution, colour-coded by education level.
K. Correlation heatmap — Likert scores
How this was calculated: Pearson r computed pairwise. * p<0.05 ** p<0.01. Scale −1 (blue) to +1 (red).
